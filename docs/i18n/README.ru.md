# groundtruth

> **Коротко** — ИИ говорит _«Готово! Добавил X, исправил Y, написал тесты.»_ groundtruth сверяет каждое утверждение с реальным diff и помечает те, которых не было. Одна команда: `npx @twarc_net/groundtruth install`.

**Узнайте, когда ваш ИИ-ассистент заявляет о работе, которую на самом деле не сделал.**

Агент завершает ход словами _«Готово! Добавил middleware `rateLimiter` в `src/server.ts`, исправил баг с таймаутом и добавил тесты.»_ Вы доверяете сводке, делаете коммит и идёте дальше. Через две недели продакшен падает — rate limiter так и не был написан. groundtruth читает сводку, извлекает каждое конкретное утверждение и сверяет его с тем, что действительно изменилось — с **истиной** (ground truth).

```text
groundtruth — claim check

  ❌ unsupported  symbol `rateLimiter`
  ❌ unsupported  file src/server.ts
  ❌ unsupported  tests

  3 claims · 0 verified · 3 unsupported
```

> Всё изменение выше было одной правкой README. groundtruth поймал все три ложных утверждения.

## Зачем это нужно

«Фантомные изменения» — работа, о которой сводка заявляет, но которую так и не реализует — самый частый вид несоответствий у ИИ-агентов. Тесты ловят _неправильный_ код; никто не ловит код, который просто _никогда не был написан_. Принцип один: **diff не врёт.**

## Попробуйте за 30 секунд

```bash
npx @twarc_net/groundtruth verify --transcript examples/phantom-change.jsonl --no-git
```

## Установка

Требуется Node ≥ 20. Глобальная установка не нужна — хук запускается через `npx`.

```bash
# Установить как Stop-хук Claude Code для этого проекта
npx @twarc_net/groundtruth install

# …или для всех проектов
npx @twarc_net/groundtruth install --global
```

Перезапустите Claude Code (или выполните `/hooks`), и groundtruth будет проверять каждый ход автоматически.

## Как это работает

Читает ход → собирает доказательства из вызовов инструментов и git diff → извлекает утверждения из сводки → проверяет каждое и выносит вердикт:

| Вердикт | Значение |
|---|---|
| ✅ **verified** | Утверждение подтверждено конкретными доказательствами. |
| ❌ **unsupported** | Утверждение проверяемо, но доказательств **нет** — фантомное изменение. |
| ⚠️ **review** | Смысловое или неоднозначное (напр. _«исправил баг»_); показывается для внимания, никогда не считается провалом. |

Консервативен по замыслу: помечает **unsupported** только когда утверждение явно проверяемо и ничто его не подтверждает — лучше пропустить, чем обвинить напрасно.

## Честные ограничения

Он проверяет, что заявленная работа **присутствует в diff**, а не что она **корректна** — для этого есть тесты.
